# stdbuf 命令

## 命令概述

`stdbuf` 命令用于以修改后的缓冲方式运行程序，可以控制标准输入、标准输出和标准错误三个 I/O 流的缓冲行为。它基于 GNU C 库的 `LD_PRELOAD` 机制工作。

**命令格式：**
```
stdbuf 选项... 命令 [参数]...
```

> **限制：** 命令必须使用 ISO C 的 `FILE` 流进行输入/输出，且不自行调整标准流的缓冲。例如 `dd` 不使用 `FILE` 流，`tee` 会自行调整流缓冲，因此 `stdbuf` 对它们无效。

## 完整参数列表

| 短选项 | 长选项 | 说明 |
|--------|--------|------|
| `-i` | `--input=MODE` | 调整标准输入流的缓冲模式 |
| `-o` | `--output=MODE` | 调整标准输出流的缓冲模式 |
| `-e` | `--error=MODE` | 调整标准错误流的缓冲模式 |

## 参数详细说明

### `-i, --input=MODE`
调整标准输入流的缓冲模式。

### `-o, --output=MODE`
调整标准输出流的缓冲模式。

### `-e, --error=MODE`
调整标准错误流的缓冲模式。

### MODE 缓冲模式

| 模式 | 说明 |
|------|------|
| `L` | 行缓冲模式。数据在输出换行符或从终端设备读取输入时被合并。**不能用于标准输入** |
| `0` | 禁用缓冲。数据立即输出，输入只读取请求的数量。禁用输入缓冲不一定影响流输入函数的响应性或阻塞行为 |
| `SIZE` | 指定全缓冲模式的缓冲区大小。SIZE 可以是整数，后跟可选的乘法后缀 |

### SIZE 后缀

| 后缀 | 含义 |
|------|------|
| `KB` | 1000（千字节） |
| `K` | 1024（Kibi 字节） |
| `MB` | 1000×1000（兆字节） |
| `M` | 1024×1024（Mebi 字节） |
| `GB` | 1000×1000×1000（吉字节） |
| `G` | 1024×1024×1024（Gibi 字节） |
| `T`, `P`, `E`, `Z`, `Y`, `R`, `Q` | 类推，同时支持二进制前缀（如 `KiB`=K, `MiB`=M） |

## 使用示例

1. **将标准输出设为行缓冲**
   ```bash
   stdbuf -oL tail -f /var/log/syslog
   ```

2. **禁用标准输出缓冲（实时输出）**
   ```bash
   stdbuf -o0 long_running_script.sh
   ```

3. **设置标准输出缓冲区为 4KB**
   ```bash
   stdbuf -o4K mycommand
   ```

4. **同时设置多个流的缓冲**
   ```bash
   stdbuf -oL -e0 mycommand
   ```

5. **用于实时查看管道输出**
   ```bash
   stdbuf -oL ./generate_output | grep "pattern"
   ```

6. **设置输入流缓冲区大小**
   ```bash
   stdbuf -i8K mycommand < input_file
   ```

## WinuxCmd 实现状态

**已实现**

当前实现只调整父进程侧缓冲并启动子进程。GNU `stdbuf` 使用 `LD_PRELOAD` 机制将缓冲策略更准确地作用到被执行程序本身，这在 Windows 上需要不同的实现方式。
