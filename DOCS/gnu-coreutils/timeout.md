# timeout 命令

## 命令概述

`timeout` 命令用于运行一个有时间限制的命令。如果命令在指定时间内未完成，`timeout` 会向其发送信号终止它。

**命令格式：**
```
timeout [选项]... 时长 命令 [参数]...
```

## 完整参数列表

| 短选项 | 长选项 | 说明 |
|--------|--------|------|
| `-s` | `--signal=SIGNAL` | 超时后发送的信号（默认 TERM） |
| `-k` | `--kill-after=DURATION` | 发送初始信号后再等指定时长发送 KILL |
| `-f` | `--foreground` | 不创建独立的后台进程组，允许命令使用前台 TTY |
| `-p` | `--preserve-status` | 超时时返回被执行命令的退出状态 |
| `-v` | `--verbose` | 超时时在标准错误输出诊断信息 |

## 参数详细说明

### `-s, --signal=SIGNAL`
超时后发送给命令的信号。可以是信号名称（如 `HUP`、`TERM`、`KILL`）或信号编号（如 `1`、`9`、`15`）。默认信号是 `TERM`。

### `-k, --kill-after=DURATION`
在发送初始信号后，等待指定时长，如果命令仍未终止则发送 `KILL` 信号。时长从 `timeout` 发送初始信号的时刻开始计算。如果主时长或此选项指定的时长为 0，则此选项无效。

### `-f, --foreground`
不创建独立的后台进程组，使被管理的命令可以正常使用前台 TTY。适用于以下场景：
- 命令是交互式的，需要从终端读取输入
- 用户希望直接从终端向命令发送信号（如 Ctrl-C）

在此模式下，命令的子进程不会被超时控制，且不会向命令发送 SIGCONT。

### `-p, --preserve-status`
超时时返回被执行命令的退出状态，而不是表示超时的特定退出状态。适用于被管理命令支持不确定运行时长的情况。

### `-v, --verbose`
当超时发生时，在标准错误输出诊断信息，说明发送了什么信号。

### DURATION 时长格式
时长是一个浮点数，后跟可选的单位：

| 后缀 | 单位 |
|------|------|
| `s` | 秒（默认） |
| `m` | 分钟 |
| `h` | 小时 |
| `d` | 天 |

时长为 0 表示禁用关联的超时。十六进制数只有在带有 `p` 风格指数时才能与 `d` 后缀一起使用，如 `0x1p0d` 表示一天。实际超时时长取决于系统条件，指定亚秒级超时时应特别注意。

## 使用示例

1. **限制命令运行时间**
   ```bash
   timeout 30 long_running_command
   ```

2. **使用分钟为单位**
   ```bash
   timeout 5m make build
   ```

3. **发送指定信号**
   ```bash
   timeout -s INT 5 sleep 20
   ```

4. **超时后强制杀死**
   ```bash
   timeout -s TERM -k 10s 60s mycommand
   # 60秒后发送 TERM，再等10秒后发送 KILL
   ```

5. **保留命令的退出状态**
   ```bash
   timeout -p 10s mycommand
   ```

6. **前台模式运行交互式命令**
   ```bash
   timeout -f 60s bash
   ```

7. **显示超时诊断信息**
   ```bash
   timeout -v -s KILL 5s slow_command
   ```

8. **使用浮点数指定亚秒超时**
   ```bash
   timeout 0.5s quick_task
   ```

9. **禁用超时（时长为 0）**
   ```bash
   timeout 0s mycommand
   ```

## 退出状态

| 退出码 | 含义 |
|--------|------|
| `124` | 命令超时（未指定 `--preserve-status`） |
| `125` | `timeout` 本身失败 |
| `126` | 命令已找到但无法调用 |
| `127` | 命令未找到 |
| `137` | 命令或 `timeout` 收到 KILL(9) 信号（128+9） |
| 其他 | 命令的退出状态 |
