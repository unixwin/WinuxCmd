# nproc 命令

## 命令概述

`nproc` 命令用于打印可用的处理器数量。这对于需要知道可以启动多少并行进程的脚本非常有用。

**命令格式：**
```
nproc [选项]...
```

## 完整参数列表

| 短选项 | 长选项 | 说明 |
|--------|--------|------|
| | `--all` | 打印已安装的处理器总数 |
| | `--ignore=N` | 如果可能，忽略 N 个处理器 |
| | `--help` | 显示帮助信息并退出 |
| | `--version` | 输出版本信息并退出 |

## 参数详细说明

### `--all`
打印系统中已安装的处理器总数，而不是当前可用的处理器数量。在某些系统上，某些处理器可能被离线或被系统保留。

### `--ignore=N`
在计算可用处理器数量时，忽略 N 个处理器。这在你想为系统保留一些处理器时很有用。

例如，如果系统有 8 个处理器：
- `nproc` 输出 `8`
- `nproc --ignore=2` 输出 `6`

如果忽略的数量大于或等于可用处理器数量，则输出 `1`（至少保留一个处理器）。

## 使用示例

1. **显示可用处理器数量**
   ```bash
   nproc
   ```
   输出示例：`8`

2. **显示所有已安装的处理器**
   ```bash
   nproc --all
   ```

3. **为系统保留 2 个处理器**
   ```bash
   nproc --ignore=2
   ```
   如果系统有 8 个处理器，输出：`6`

4. **在 Make 中使用 nproc 进行并行编译**
   ```bash
   make -j$(nproc)
   ```

5. **在 CMake 中使用 nproc**
   ```bash
   cmake --build . -j$(nproc)
   ```

6. **为系统保留一半处理器用于其他任务**
   ```bash
   nproc --ignore=$(( $(nproc) / 2 ))
   ```

7. **在并行下载中使用**
   ```bash
   aria2c -j$(nproc) https://example.com/files.txt
   ```

8. **显示帮助信息**
   ```bash
   nproc --help
   ```

9. **显示版本信息**
   ```bash
   nproc --version
   ```

## 常见用法

`nproc` 常用于以下场景：

- **并行编译**：`make -j$(nproc)` 使用所有可用核心进行编译
- **并行处理**：确定可以启动多少个工作进程
- **资源分配**：在容器或虚拟化环境中检查可用资源
- **脚本自动化**：根据 CPU 核心数动态调整任务并行度
