# kill 命令

## 命令概述

`kill` 命令用于向进程发送信号，使其终止或以其他方式响应信号。也可以列出可用的信号信息。

**命令格式：**
```
kill [-s 信号 | --signal 信号 | -信号] PID...
kill [-l | --list | -t | --table] [信号]...
```

> **注意：** 由于 shell 别名和内建 `kill` 函数，在交互式 shell 或脚本中直接使用 `kill` 可能会得到与此处描述不同的功能。通过 `env` 调用（即 `env kill ...`）可以避免 shell 干扰。

## 完整参数列表

| 短选项 | 长选项 | 说明 |
|--------|--------|------|
| `-s` | `--signal=SIGNAL` | 指定要发送的信号名称或编号 |
| `-l` | `--list` | 列出所有信号名称，或将信号编号转换为名称 |
| `-t` | `--table` | 以表格形式列出信号编号、名称和描述 |

## 参数详细说明

### `-s, --signal=SIGNAL`
指定要发送的信号。可以是信号名称（如 `HUP`、`TERM`、`KILL`）或信号编号（如 `1`、`9`、`15`）。默认信号是 `TERM`。

### `-l, --list`
列出支持的信号名称，每行一个。如果指定了信号参数，则在信号名称和信号编号之间进行转换。

### `-t, --table`
与 `--list` 类似，但以表格形式输出信号编号、名称和描述。

### PID 参数

| PID 值 | 含义 |
|--------|------|
| 正数 | 向进程 ID 为 PID 的进程发送信号 |
| `0` | 向当前进程组中的所有进程发送信号 |
| `-1` | 向用户有权限发送信号的所有进程发送信号 |
| 小于 -1 | 向进程组 ID 等于 PID 绝对值的所有进程发送信号 |

如果需要将负的 PID 作为第一个参数，应在其前加 `--`。但作为 POSIX 的常见扩展，`kill -signal -pid` 形式不需要 `--`。

### 特殊信号 0
信号编号 `0` 不表示有效信号，但可用于测试 PID 参数指定的进程是否可以接收信号。

## 使用示例

1. **发送默认 TERM 信号终止进程**
   ```bash
   kill 1234
   ```

2. **发送指定信号**
   ```bash
   kill -s KILL 1234
   ```

3. **使用信号编号**
   ```bash
   kill -9 1234
   ```

4. **列出所有信号名称**
   ```bash
   kill -l
   ```

5. **以表格形式列出信号**
   ```bash
   kill -t
   ```

6. **将信号编号转换为名称**
   ```bash
   kill -l 9
   # 输出: KILL
   ```

7. **将信号名称转换为编号**
   ```bash
   kill -l TERM
   # 输出: 15
   ```

8. **向当前进程组的所有进程发送信号**
   ```bash
   kill -s TERM 0
   ```

9. **向指定进程组发送信号**
   ```bash
   kill -s HUP -1234
   ```

10. **测试进程是否存在**
    ```bash
    kill -0 1234 && echo "Process exists"
    ```

11. **使用 SIG 前缀**
    ```bash
    kill -s SIGTERM 1234
    ```

## 常用信号

| 信号 | 编号 | 说明 |
|------|------|------|
| `HUP` | 1 | 挂起信号 |
| `INT` | 2 | 中断信号（Ctrl-C） |
| `QUIT` | 3 | 退出信号（Ctrl-\） |
| `KILL` | 9 | 强制杀死信号（不可捕获） |
| `TERM` | 15 | 终止信号（默认） |
| `STOP` | 19 | 停止信号（不可捕获） |
| `CONT` | 18 | 继续信号 |
| `TSTP` | 20 | 终端停止信号（Ctrl-Z） |
