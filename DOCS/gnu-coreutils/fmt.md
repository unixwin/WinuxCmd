# fmt 命令

## 命令概述

`fmt` 命令用于将文本段落格式化为指定的宽度，使其更易于阅读。它从指定的文件或标准输入读取内容，并将文本重新格式化为适合终端或文档显示的宽度。

**命令格式：**
```
fmt [选项]... [文件]...
```

## 完整参数列表

| 短选项 | 长选项 | 说明 |
|--------|--------|------|
| `-c` | `--crown-margin` | 保留前两行的缩进 |
| `-t` | `--tagged-paragraph` | 保留第一行的缩进 |
| `-s` | `--split-only` | 只在必要时断行，不合并行 |
| `-u` | `--uniform-spacing` | 使用统一的间距（单词间一个空格，句子间两个空格） |
| `-w` | `--width=宽度` | 设置输出行的最大宽度（默认 75 列） |
| `-p` | `--prefix=前缀` | 只处理以指定前缀开头的行 |
| `-g` | `--goal=宽度` | 设置目标宽度（默认比最大宽度小 3%） |
| | `--help` | 显示帮助信息 |
| | `--version` | 显示版本信息 |

## 参数详细说明

### `-c, --crown-margin`
保留段落前两行的缩进。第一行的缩进成为整个段落的缩进，第二行的缩进成为后续行的缩进。

### `-t, --tagged-paragraph`
保留第一行的缩进，但后续行的缩进可能不同。类似于 `-c`，但适用于缩进不同的情况。

### `-s, --split-only`
只在必要时断行，不合并短行。这可以防止将短行合并到前一行中。

### `-u, --uniform-spacing`
使用统一的间距规则：单词之间使用一个空格，句子之间使用两个空格。

### `-w, --width=宽度`
设置输出行的最大宽度。如果未指定，默认为 75 列。可以使用 `=` 或空格分隔选项和宽度值。

### `-p, --prefix=前缀`
只处理以指定前缀开头的行，其他行保持不变。这对于处理注释或标记的文本非常有用。

### `-g, --goal=宽度`
设置目标宽度。格式化算法会尝试使行接近这个宽度，但不会超过最大宽度。默认比最大宽度小 3%。

### `--help`
显示帮助信息并退出。

### `--version`
显示版本信息并退出。

## 使用示例

1. **基本格式化**
   ```bash
   fmt file.txt
   ```
   将文件内容格式化为 75 列宽度。

2. **设置特定宽度**
   ```bash
   fmt -w 60 file.txt
   ```
   将文件内容格式化为 60 列宽度。

3. **保留段落缩进**
   ```bash
   fmt -c file.txt
   ```
   格式化文本并保留段落前两行的缩进。

4. **处理带前缀的行**
   ```bash
   fmt -p "> " email.txt
   ```
   只格式化以 "> " 开头的行（如邮件引用）。

5. **只断行不合并**
   ```bash
   fmt -s file.txt
   ```
   只在必要时断行，不合并短行。

6. **统一间距**
   ```bash
   fmt -u file.txt
   ```
   使用统一的间距规则格式化文本。

7. **从标准输入格式化**
   ```bash
   echo "This is a long line that needs to be formatted properly for better readability." | fmt -w 40
   ```
   将输入文本格式化为 40 列宽度。

8. **组合选项**
   ```bash
   fmt -w 80 -c -u file.txt
   ```
   设置 80 列宽度，保留段落缩进，使用统一间距。
